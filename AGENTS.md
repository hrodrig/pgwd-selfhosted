# Agent Guidelines (pgwd-selfhosted)

- Use **English** for all project artifacts (code, comments, commit messages, docs, README).
- **Scope:** **pgwd-selfhosted** owns deployment (Helm, Compose, **`run/`**, chart CI). **pgwd** owns the Go app and releases only — the Helm chart is **not** on **`pgwd`** **`main`**; all chart work stays in **`run/kubernetes/helm/pgwd/`** here.
- Follow **git flow**: work on `develop`; **`main`** for production snapshots; **annotated tags** `v<semver>` on `main` for infra releases (see root **`VERSION`**).
- **`VERSION`** (repository root): canonical **pgwd-selfhosted** semver (`0.1.8` style, no `v`). When it changes, align the README **Version** badge, optional CHANGELOG, and Git tag **`v…`** on **`main`**. **`Chart.yaml` `version:`** tracks the **Helm chart package** only — bump it when **`run/kubernetes/helm/pgwd/`** changes materially, **not** automatically on every **`VERSION`** bump (see **Versioning** in README and **`.cursor/rules/version-sync.mdc`**).
- **`PGWD_VERSION`** in **`${PGWD_HOST_DATA}/.env`** (recommended) or any env file passed to Compose: pins the **application** OCI image (`ghcr.io/hrodrig/pgwd:…`); align with **[pgwd](https://github.com/hrodrig/pgwd)** releases — not the same field as this repo’s **`VERSION`**. Docs and chart defaults track **v0.6.4+** features (optional **SQLite**, **HTTP `/metrics`**, hysteresis, **`databases:`**); see upstream README — **pgwd-selfhosted** does not ship Prometheus/Grafana.
- **pgwd** uses **`make release-check`** (Go lint, tests, Docker integration/e2e, image scan). **This repo** has no Go tests; **`make release-check`** here means **`helm lint`**, **`helm template`** + **kubeconform** (same scenarios as [**.github/workflows/helm-lint.yml**](.github/workflows/helm-lint.yml)), and **`docker compose … config`** for **minimal** (dummy **`PGWD_DB_URL`**). Requires **helm**, **kubeconform**, and **docker** on **`PATH`**. Heavier checks: **`make test-helm-kind`**, **`make test-compose-platforms`** (Ansible — **`testing/platforms/README.md`**). Bare **`make`** lists targets.
- Keep **`run/`** paths, **`PGWD_HOST_DATA`**, and **`${PGWD_HOST_DATA}/.env`** documentation consistent across README files (always **`--env-file`** from the clone root — no default of secrets in the repository root).
- **pgwd multi-database:** When documenting or editing **`config.extra`** / YAML with **`databases:`**, align with upstream **[Multi-database limitations](https://github.com/hrodrig/pgwd/blob/main/README.md#multi-database-limitations)** — no **`databases:`** with **`kube.postgres`** in the same process; unique **`client` per `databases:`** entry when the same DB name exists on different hosts (SQLite key omits URL host).
- Do not commit without first showing the proposed commit message and getting **explicit user approval** (same convention as **pgwd**).
